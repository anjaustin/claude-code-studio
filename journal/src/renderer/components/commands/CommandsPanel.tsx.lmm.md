# LMM: src/renderer/components/commands/CommandsPanel.tsx

## RAW

This 279-line file is the largest in the cluster by far and contains two distinct concerns: (1) a static catalog of slash commands and keyboard shortcuts (lines 9-69), and (2) a 3-tab UI to browse them (lines 87-277). The catalog is a `Record<string, CommandEntry[]>` grouped by category ("Model & Effort", "Session", "Workflow", "Config", "Info & Utils") plus a flat `KEYBOARD_SHORTCUTS` array. The component exports `CommandsPanel` accepting `onSendCommand: (command: string) => void` (line 73) which is invoked when the user clicks any command — the command is fed back into the terminal via the `sendRef` plumbing from TerminalPanel.

The tab bar (lines 105-134) is a segmented control with three tabs: Quick Actions (delegates to `<QuickCommands>`), All Commands (accordion of categorized command sections, lines 138-234), and Shortcuts (a flat table-like list of keybindings, lines 236-275). The "All Commands" accordion uses `expandedSection: string | null` to allow only one open section at a time (line 79). Each command button extracts the command name before the first space (line 190: `cmd.name.split(' ')[0]`) so `'/model [model]'` is sent as `/model` — clever but lossy: the user can't actually pass an argument from this UI.

Open questions:
- `cmd.name.split(' ')[0]` (line 190) silently drops arguments. Why not let the user click through to an input prompt that fills the placeholder?
- The catalog is hardcoded in JSX — should it live in a JSON config that can be updated without a rebuild?
- The hover style on command rows uses inline `e.currentTarget.style.background = ...` (lines 204-209) — imperative DOM manipulation inside React. Why not the `useState(hovered)` pattern used elsewhere?
- `KEYBOARD_SHORTCUTS` are documentation only — the app doesn't actually bind any of them. Misleading?
- The accordion is one-at-a-time (line 155); a developer browsing commands might want to see multiple sections.

## NODES

1. **Hardcoded slash-command catalog** (lines 9-55) — 30+ entries; updating means editing this file vs editing a config.
2. **Lossy command extraction** (line 190) — `split(' ')[0]` discards arguments; user clicks "/model [model]" and sends bare "/model".
3. **Imperative hover** (lines 204-209) — direct DOM mutation; inconsistent with React-state hover used in TitleBar, Sidebar, QuickCommands, SettingsPanel.
4. **Shortcut list is documentation only** — no `useEffect` that binds these to actual keyboard handlers; user might infer the app implements them.
5. **One-at-a-time accordion** (line 155) — only one section open; mass scanning requires repeated clicks.
6. **Inline `SectionHeader` pattern** (lines 89-103) — fifth copy in the cluster.
7. **Tabs are buttons but lack `role="tab"`** (lines 115-133) — semantically a tablist; SR users see three unrelated buttons.
8. **Tab content lacks `role="tabpanel"`** — pairing absent.
9. **No `aria-expanded` on accordion buttons** (line 153) — visual chevron rotates but no SR announcement.
10. **Magic 5-px and 4-px paddings** scattered (lines 110, 138, 186) — likely intentional design; could be tokens.
11. **`<kbd>` element used correctly** (line 257) — small win for the Shortcuts tab.
12. **`#fff` literal for active tab text** (line 124) — vs `var(--text-on-accent)` would be theme-aware.
13. **Description text `text-align: right`** (line 222) — readability suffers when the value is long.

Tensions:
- **T1: Catalog-as-code vs catalog-as-data.** Hardcoded JSX is fast to edit but hard to maintain at scale; a JSON config + autogenerated UI scales but adds boilerplate.
- **T2: Inform vs execute.** The shortcuts tab lists keys the app does not bind. The slash commands tab sends the bare command without args. Both leak intent: the UI shows what you *could* do, not what you *can* do here.
- **T3: Two hover-state patterns in one file.** Imperative on accordion rows (line 204), implicit nowhere — but other panels (QuickCommands, Sidebar) use React state. Inconsistency creates confusion.

## REFLECT

**Core insight:** CommandsPanel is a docs-disguised-as-UI: it shows command names and descriptions but its only real interaction is "send the bare command string," which makes the parameterized commands (most of them) inert; it needs to either commit to argument prompts or admit it's a reference card.

Resolved tensions:
- **T1 resolved:** Move catalog to `src/shared/commands.ts` exporting typed `SLASH_COMMANDS` and `KEYBOARD_SHORTCUTS`. CommandsPanel imports. QuickCommands' separate inline catalog (QuickCommands.tsx lines 10-33) can deduplicate against this.
- **T2 resolved:** For parameterized commands, render a small inline input that the user fills before sending. For unbound shortcuts, add a "Reference only — bind in terminal" footnote or remove non-applicable entries.
- **T3 resolved:** Standardize on React-state hover (matches QuickCommands.tsx, SettingsPanel.tsx). Replace lines 204-209 with `useState`-based hover or extract a `useHover()` hook.

Hidden assumptions:
- That clicking "/model" without an argument is useful (it likely shows current model — but the user is told nothing).
- That listing shortcuts the app does not bind is harmless documentation (it implies they work in this surface).
- That one-at-a-time accordion is the desired UX (vs multi-open).

## SYNTHESIZE

**What it should become:**
- Extract `SLASH_COMMANDS` and `KEYBOARD_SHORTCUTS` to `src/shared/commands.ts`.
- For commands with placeholders like `[model]`, show an inline `<input>` that becomes the argument when present.
- Tablist semantics: `role="tablist"` on container, `role="tab"` + `aria-selected` + `aria-controls` on each tab, `role="tabpanel"` + `aria-labelledby` on content.
- Accordion: `aria-expanded` on the toggle button.
- Allow multi-open accordion (set of open section names, not single string).
- Replace imperative hover with React-state.
- Extract `PanelHeader` (lines 89-103).

Actionable items:
1. Lift `SLASH_COMMANDS` and `KEYBOARD_SHORTCUTS` to `src/shared/commands.ts`. Import from both CommandsPanel and QuickCommands.
2. Add `role="tablist"` / `role="tab"` / `aria-selected={tab === t.id}` to the tab buttons (lines 114-133).
3. Add `aria-expanded={isOpen}` and `aria-controls={section + '-panel'}` to the accordion button (line 153).
4. Replace `onMouseEnter`/`onMouseLeave` inline DOM mutation (lines 204-209) with a `hoveredCmd` state and conditional `background`.
5. Render an inline `<input>` for commands whose `name` contains `[...]` placeholders; on Enter, send `name.split(' ')[0] + ' ' + inputValue`.
6. Replace `#fff` (line 124) with `var(--text-on-accent)` token.

Risks:
- Adding inline inputs for argumented commands is a significant UX expansion — design first, then build.
- Moving catalog to shared may surface that QuickCommands and CommandsPanel duplicate concepts under different names ("Opus" vs "/model opus"); reconcile carefully.
- Multi-open accordion changes the perceived "calm" of the panel; consider a "collapse all" button.
